# SRS - ระบบวัดระดับความขุ่นของน้ำ

## 1. ข้อมูลทั่วไป
- ชื่อระบบ: ระบบวัดระดับความขุ่นของน้ำ
- วัตถุประสงค์: เพื่อช่วยติดตามระดับความขุ่นของน้ำและแจ้งเตือนเมื่อค่าความขุ่นสูงเกินกำหนด

## 2. วัตถุประสงค์ของระบบ
ระบบนี้มีวัตถุประสงค์เพื่อ
- ตรวจวัดและแสดงระดับความขุ่นของน้ำ
- แจ้งเตือนเมื่อถึงกำหนดหรือพบว่าความขุ่นสูงเกินระดับที่กำหนด
- บันทึกประวัติการล้างถังและงานซ่อมบำรุง
- รองรับการใช้งานทั้งบนคอมพิวเตอร์และมือถือ

## 3. ขอบเขตของระบบ
### 3.1 ฟังก์ชันหลัก
- แสดงระดับความขุ่นของน้ำในรูปแบบที่เข้าใจง่าย
- แจ้งเตือนเมื่อความขุ่นเกินค่าที่กำหนด
- บันทึกข้อมูลประวัติการล้างถัง
- บันทึกข้อมูลประวัติการซ่อมบำรุง

### 3.2 ผู้ใช้งาน
- ผู้ดูแลระบบ / เจ้าหน้าที่ที่รับผิดชอบการตรวจสอบน้ำ

## 4. ความต้องการเชิงฟังก์ชัน (Functional Requirements)
1. ระบบต้องแสดงระดับความขุ่นของน้ำให้ผู้ใช้เห็นได้ชัดเจน
2. ระบบต้องแจ้งเตือนเมื่อระดับความขุ่นถึงเกณฑ์ที่กำหนด
3. ระบบต้องบันทึกประวัติการล้างถัง
4. ระบบต้องบันทึกประวัติการซ่อมบำรุง
5. ระบบต้องรองรับการใช้งานบนคอมพิวเตอร์และมือถือ

## 5. ความต้องการเชิงไม่ใช่ฟังก์ชัน (Non-Functional Requirements)
- ระบบต้องออกแบบให้แยกส่วน Frontend และ Backend อย่างชัดเจน
- ระบบต้องใช้งานง่ายและเข้าใจได้ง่าย
- ระบบต้องมีความเสถียรในการแสดงข้อมูลและแจ้งเตือน

## 6. ข้อกำหนดเพิ่มเติม
- ระบบควรมีการจัดการข้อมูลที่เป็นระเบียบและสามารถตรวจสอบย้อนหลังได้
- ระบบควรมีความปลอดภัยในการเข้าถึงข้อมูลสำหรับผู้ที่มีสิทธิ์ใช้งานเท่านั้น

## 7. สรุป
ระบบนี้เป็นระบบที่ช่วยในการติดตามและจัดการระดับความขุ่นของน้ำ โดยมีฟังก์ชันหลักเพื่อแจ้งเตือนและบันทึกประวัติการบำรุงรักษา เพื่อให้ผู้ดูแลสามารถจัดการงานได้อย่างมีประสิทธิภาพ
