# CLAUDE.md

## คำแนะนำสำหรับการพัฒนาโครงการ

### 1. เป้าหมายของโครงการ
โครงการนี้เป็นระบบเว็บสำหรับวัดระดับความขุ่นของน้ำ โดยมีวัตถุประสงค์เพื่อ
- ตรวจสอบและแสดงระดับความขุ่นของน้ำ
- แจ้งเตือนเมื่อระดับความขุ่นสูงเกินค่าที่กำหนด
- บันทึกประวัติการล้างถังและงานซ่อมบำรุง
- รองรับการใช้งานทั้งบนคอมพิวเตอร์และมือถือ

### 2. หลักการพัฒนา
- แยกส่วน Frontend และ Backend อย่างชัดเจน
- ออกแบบระบบให้ใช้งานง่ายและเข้าใจง่าย
- เน้นความถูกต้องของข้อมูลและความเสถียรของการแจ้งเตือน
- ควรเก็บข้อมูลประวัติอย่างเป็นระเบียบเพื่อการตรวจสอบย้อนหลัง

### 3. ข้อควรพิจารณาเมื่อแก้ไขโค้ด
- ตรวจสอบให้แน่ใจว่าเปลี่ยนแปลงไม่กระทบต่อฟังก์ชันหลักของระบบ
- ถ้ามีการเพิ่มฟีเจอร์ใหม่ ควรอัปเดตเอกสาร SRS และ README ให้สอดคล้องกัน
- ควรเขียนคำอธิบายหรือคอมเมนต์ที่ชัดเจนในโค้ดที่ซับซ้อน
- เลือกชื่อตัวแปรและฟังก์ชันที่สื่อความหมายได้ชัดเจน

### 4. เอกสารที่ควรอ้างอิง
- SRS.md สำหรับรายละเอียดความต้องการระบบ
- README.md สำหรับภาพรวมและคำแนะนำการใช้งาน

### 5. คำแนะนำทั่วไป
- ให้ความสำคัญกับความน่าเชื่อถือของระบบและการแจ้งเตือนที่แม่นยำ
- พัฒนาระบบให้สามารถขยายได้ในอนาคต
- ใช้แนวทางที่ง่ายต่อการบำรุงรักษาและพัฒนาเพิ่มเติม
